# ミッション
TalkCapitalに**ストリーミングモード**を追加する。
講演中にリアルタイム文字起こしし、一定の区切りでグラレコPNGを更新、講演終了時に最終版を生成するWebサービス。

## アーキテクチャ概要
```
ブラウザ (マイク) → PCM 16kHz via WebSocket → Node.js サーバー
  ├→ Amazon Transcribe Streaming（リアルタイム文字起こし）
  │    └→ 確定テキスト蓄積
  ├→ 500文字蓄積ごとに:
  │    Bedrock構造化 → Excalidraw JSON → PNG → WebSocketでブラウザに送信
  └→ 講演終了時:
       全文字起こし → Bedrock最終構造化 → 最終版PNG
```

# 実装フェーズ（9フェーズ、この順で実装する）

## Phase 0: 依存パッケージ追加 [S]
- `npm install @aws-sdk/client-transcribe-streaming ws express`
- `npm install -D @types/ws @types/express`
- `package.json` の scripts に `"serve": "node dist/server.js"` を追加
- `npm run build && npm run lint && npm run test` が引き続き成功すること

## Phase 1: 型定義 [S]
- `src/types/structured-content.ts` に `interimStructuredContentSchema` を追加
  - blocks min(1), speechBubbles min(0), actions min(0).max(3)
  - 既存の `structuredContentSchema` は変更しない
- `src/types/ws-messages.ts` を新規作成（WebSocketメッセージ型定義）

## Phase 2: Transcribe Streaming ラッパー [M]
- `src/streaming/transcribe-stream.ts` を新規作成
  - `@aws-sdk/client-transcribe-streaming` の `StartStreamTranscriptionCommand` を使用
  - PCMチャンクを `PassThrough` ストリーム経由でTranscribeに供給
  - partial/final results をEventEmitterで通知
  - 10秒間隔で無音チャンク送信（15秒タイムアウト防止）
- `test/streaming/transcribe-stream.test.ts` を新規作成（モックテスト）

## Phase 3: PlaywrightPool [M] ※Phase 2と並列可
- `src/streaming/playwright-pool.ts` を新規作成
  - サーバー起動時に `chromium.launch()` を1回だけ実行しブラウザを保持
  - `export/index.html` を事前ロードしたPageを保持
  - `exportToPng(doc, scale)` は `page.evaluate()` のみ（起動オーバーヘッドなし）
  - busy排他制御（同時レンダリング防止）
  - 50回レンダリングごとにページ再作成（メモリリーク防止）
- `test/streaming/playwright-pool.test.ts` を新規作成（モックテスト）

## Phase 4: StreamingSession [L]
- `src/streaming/session.ts` を新規作成（セッション管理の中核）
  - `start()` → Transcribeストリーミング開始
  - `feedAudio(chunk)` → PCMをTranscribeに供給
  - final result受信時、蓄積テキスト +500文字超で構造化+PNG生成をトリガー
  - 構造化中に次のトリガーが来たらスキップ（完了後に最新テキストで再実行）
  - `end()` → 全文字起こしで最終版生成（既存の `structureTranscript()` + 厳格スキーマ）
  - ストリーミング用システムプロンプト（途中テキスト前提）を定義
  - `interimToStructured()` アダプタ: 不足分をプレースホルダ補完して `renderToExcalidraw()` に渡す
- `test/streaming/session.test.ts` を新規作成（モックテスト）

## Phase 5: サーバー + WebSocket [M]
- `src/server.ts` を新規作成

  - `ws` ライブラリ: WebSocket接続管理

    - サーバー→クライアント: `transcript_partial/final`, `graphic_update/final`, `status`, `error`

# GitHub認証（必須・起動直後に実行）
- 作業開始時に `.env.local` から `GITHUB_TOKEN` を読み込み、`GH_TOKEN` として export すること:
  ```bash
  export GH_TOKEN=$(grep '^GITHUB_TOKEN=' .env.local | cut -d'=' -f2)
  ```
- 以降の Issue 作成・更新・クローズ、PR 作成・マージ等の GitHub 操作はすべて `gh` CLI 経由で行うこと
- git push にPATが必要な場合は以下の形式を使うこと:
  ```bash
  git push "https://x-access-token:${GH_TOKEN}@github.com/o-shina/talkcapital.git" <ブランチ名>
  ```
- `GH_TOKEN` の値をログ・コミットメッセージ・ExecPlan・PR本文に絶対に出力しないこと
- `.env.local` はGit管理対象外（.gitignore済み）。コミットに含めないこと
