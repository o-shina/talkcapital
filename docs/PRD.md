# TalkCapital - Product Requirements Document

## 概要

15分講演の音声を入力として、手書き風グラフィックレコーディングPNGを自動生成するCLIツール。
用途は講演向けに限定し、会議・ディスカッション用途の一般化はスコープ外。

**これは「自動お絵描き」ではない。講演メッセージを構造化された視覚資産に変換する装置である。**

---

## MVPのゴール

講演終了直後に以下を生成できること:
- 講演内容をまとめた手書き風グラレコ
- 高解像度PNG 1枚（3840x2160）

**生成時間**: 理想 1分以内 / 許容 数分以内

---

## 技術スタック（固定）

| レイヤー | 技術 |
|----------|------|
| 音声認識 | Amazon Transcribe (バッチ) |
| 構造化 | Amazon Bedrock Claude (Converse API) |
| 描画 | Excalidraw JSON 生成 |
| 出力 | Playwright ヘッドレスレンダリング → PNG |

全サービスをAWSに統一し、認証情報を一元管理する。

---

## 出力構造（固定テンプレート）

LLMに自由レイアウトさせない。以下の固定構造に必ず従う。

| # | 要素 | 制約 |
|---|------|------|
| 1 | タイトル | 講演テーマ（最大30文字） |
| 2 | メインメッセージ | 最重要1行（最大80文字） |
| 3 | コンテンツブロック | 3〜4個。各ブロック: 見出し + 箇条書き最大3 |
| 4 | 吹き出し | 最大4個（発言風） |
| 5 | 今日からできるアクション | 必ず3個（チェックボックス付き） |

**文字量制限**: 各テキスト要素は目安40文字以内。詰め込み防止、余白重視。

---

## 構造化要件（Claude）

Claude に以下の制約を与える:

1. **捏造禁止**: 話していない事実・固有の断定を足さない。不明は一般化 or 空欄
2. **各ブロック短文化**: 目安40文字以内
3. **ブロック最大4**
4. **吹き出し最大4**
5. **アクション必ず3つ**
6. **余白を意識**（情報圧縮）
7. **専門用語は講演の表現をそのまま使用**

出力は必ず構造化JSONとする（Bedrock Converse API の toolUse で強制）。

---

## 描画要件（Excalidraw）

- 手書き風スタイル（roughness: 1, fontFamily: Virgil）
- 吹き出し・付箋・矢印を活用
- レイアウトは固定テンプレートに配置（LLMは関与しない）
- 可読性優先（文字崩れ防止）
- **文字は画像生成AIを使わない**（Excalidraw テキスト要素で描画）

---

## 更新仕様

MVPでは:
- ライブ更新は Nice to have
- 本番は**講演終了後に最終清書生成**（必須）
- 秒単位更新は禁止

---

## 非機能要件

| # | 要件 | 対応 |
|---|------|------|
| 1 | ネットワーク不調でも講演成立 | 音声はローカル保存。講演後にまとめて生成可 |
| 2 | テキスト保存の最小化 | デフォルトは保存しない/短期保持 |
| 3 | 再現性重視 | テンプレ固定 + 制約 |
| 4 | 軽量設計 | 15分講演前提 |

---

## スコープ外

- 会議用途への一般化
- ブレストクラスタリング
- 画像生成AIによるポスター化
- 秒単位リアルタイム描画
- ライブストリーミング文字起こし（MVP後 Nice to have）
- Web UI（MVP後の拡張候補）
- OpenRouter対応（MVP後にGitHub Issue化）
